# Antigravity-Manager 服务器全自动部署指南 (Gemini/Claude 反代方案)

该指南详细说明了如何在服务器端通过 Docker 部署 `Antigravity-Manager`，实现 Google 模型在 OpenClaw 中的稳定反代。

## 1. 系统环境准备
如果您是在一台全新的 Ubuntu 服务器上操作，请执行以下命令安装 Docker 环境：

```bash
# 更新并安装必要的依赖
sudo apt update && sudo apt install -y curl docker.io docker-compose

# 启动并设置 Docker 开机自启
sudo systemctl enable --now docker
```

## 2. 创建部署目录与配置文件
建议将所有数据存放在 `~/antigravity` 目录下，便于管理。

```bash
# 1. 创建目录
mkdir -p ~/antigravity && cd ~/antigravity

# 2. 编写 docker-compose.yml 文件
cat <<EOD > docker-compose.yml
services:
  antigravity-manager:
    image: lbjlaq/antigravity-manager:latest
    container_name: antigravity-manager
    restart: always
    ports:
      - "8045:8045"
    environment:
      # 【必填】用于 OpenClaw 调用时的鉴权密钥，请自行修改
      - API_KEY=sk-nate-tech-2026
      # 【可选】Web 管理后台登录密码，若不设则默认使用上面的 API_KEY
      - WEB_PASSWORD=your_secure_password
      # 限制请求体大小（处理长文本/图片时有用）
      - ABV_MAX_BODY_SIZE=104857600
    volumes:
      # 持久化存储路径，确保账号登录状态不会丢失
      - ~/.antigravity_tools:/root/.antigravity_tools
EOD
```

## 3. 启动服务与日志检查
```bash
# 后台启动容器
docker-compose up -d

# 查看启动日志，确认是否成功
docker logs -f antigravity-manager
```

## 4. 🔐 核心安全操作：如何登录管理后台
容器启动后，管理后台默认运行在 `8045` 端口。**请勿在公网开放该端口。**

**推荐操作：**
1. **SSH 隧道访问**：在本地 Mac 终端运行：
   `ssh -L 8045:127.0.0.1:8045 user@your_server_ip`
2. **浏览器登录**：在 Mac 浏览器访问 `http://127.0.0.1:8045`。
3. **账号授权**：
   * 点击“账号管理”，推荐使用 **Cookie** 方式登录您的 Google 小号。
   * 配置模型映射（确保 Gemini 1.5 Pro/Flash 处于可用状态）。

## 5. 🤖 接入 OpenClaw 示例配置
在服务器的 `openclaw.json` 中的 `models.providers` 段落添加：

```json
"antigravity-local": {
  "baseUrl": "http://127.0.0.1:8045/v1",
  "apiKey": "sk-nate-tech-2026",
  "api": "openai-compatible",
  "models": [
    { "id": "gemini-3-flash", "name": "Gemini 3 Flash (AG)" },
    { "id": "gemini-3.1-pro-high", "name": "Gemini 3.1 Pro (AG)" }
  ]
}
```

## 6. 运维与风控提示
* **首选小号**：务必使用 Google 测试小号。
* **数据持久化**：所有的 Session 都存在 `~/.antigravity_tools`，请务必备份该目录。
* **反向隧道**：若 VPS IP 被封，建议通过 Tailscale 在家部署，并通过反向隧道向服务器提供接口。
