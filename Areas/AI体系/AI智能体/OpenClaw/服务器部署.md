# OpenClaw 服务器-本地 混合多 Agent 架构部署指南

[[openclaw服务器mult-agents架构图.excalidraw]]
## 1. 架构目标
实现 **“云端大脑 (Gateway) + 专家矩阵 (Multi-Agents) + 本地手脚 (Mac Node)”** 的闭环指标。
*   **云端**：负责 24/7 监听 Discord 消息、维护长期记忆和跨 Agent 调度。
*   **本地**：负责代码修改、硬件调用和 UI 交互。
*   **安全**：通过 Tailscale 隧道，实现两端在不暴露公网端口的情况下的加密互连。

## 2. 环境准备
*   **服务器 (VPS)**: 推荐 Ubuntu 22.04+，预装 Node.js 20+。
*   **本地 (Mac)**: 安装 OpenClaw CLI 并配置好开发环境。
*   **VPN**: 两端均安装 Tailscale 并登录同一账号。

## 3. 第一阶段：服务器 Gateway 部署
```bash
# 1. 安装 OpenClaw
npm install -g openclaw

# 2. 启动初始化向导
openclaw onboard
# 在向导中：绑定的 Discord Token 建议作为“CEO Muse”的角色，不设沙箱。

# 3. 开启服务器端 Gateway 守护进程
openclaw gateway start
```

## 4. 第二阶段：专家 Agent 矩阵配置
在服务器的 `~/.openclaw/openclaw.json` 中定义多个职能 Agent。

```json
{
  "agents": {
    "list": [
      { "id": "main", "name": "Muse-CEO" },
      { "id": "frontend", "name": "Frontend-Expert", "model": "anthropic/claude-3-5-sonnet" },
      { "id": "web3", "name": "Web3-Audit", "model": "google/gemini-2.0-flash" }
    ]
  },
  "bindings": [
    { "agentId": "frontend", "match": { "channel": "discord", "accountId": "frontend-lab" } }
  ]
}
```

## 5. 第三阶段：本地 Mac 节点 (Node) 挂载
在您的 **本地 Mac** 终端运行以下命令，将 Mac 变为服务器可调用的物理节点：

```bash
# 在本地 Mac 运行，给它一个名字
openclaw run --node --node-name nate-mac
```
*执行后会生成一个 \`Node URL\` 和 \`Token\`。*

## 6. 第四阶段：两端配对与验证
回到 **Discord**，对 Muse 发起配对指令：
1.  **添加节点**：\`/nodes add url:<节点URL> token:<节点Token>\`
2.  **验证能力**：\`@Muse 截一张我本地 Mac 的屏\`
3.  **协同开发**：\`@Muse 让前端工程师修改本地项目 A 的 index.tsx 的背景色\`

## 7. 运维注意事项
*   **记忆一致性**：确保服务器上的 \`MEMORY.md\` 包含项目的核心路径和规范。
*   **安全加固**：建议在 \`openclaw.json\` 中开启 \`gateway.auth.token\`，防止内网未经授权的节点接入。

## 8. 🛡️ 网络韧带：Tailscale 安全互联配置

为了在不暴露公网端口的情况下，安全地连接服务器网关和本地 Mac 节点，需要配置 Tailscale 虚拟局域网。

### 8.1 服务器端 (Linux VPS) 安装
```bash
# 下载并一键安装
curl -fsSL https://tailscale.com/install.sh | sh

# 启动并执行授权 (复制显示的链接到本地浏览器打开)
sudo tailscale up
```

### 8.2 本地端 (macOS) 安装
1. 在 Mac App Store 搜索 "Tailscale" 下载安装。
2. 菜单栏图标选择 "Sign in..." 完成授权。

### 8.3 安全与配置优化
1. **防止掉线**：在 Tailscale 管理后台 (Admin Console) 找到对应设备，点击菜单选择 **Disable key expiry**。
2. **固定内网 IP**：记录服务器的内网 IP (100.x.y.z)，后续所有 OpenClaw 跨设备调用均使用此固定 IP。

## 9. ⚙️ 配置内网 IP 互联 (Tailscale IP 应用)

通过配置 100 开头的 Tailscale 内网 IP，可以实现跨地域的静默、安全互联，无需关注公网 IP 变动。

### 9.1 本地 CLI 远程模式配置
在 **本地 Mac** 上编辑 ，使本地 CLI 指令直接操作云端网关：
```json
{
  "gateway": {
    "mode": "remote",
    "remote": {
      "url": "ws://100.x.y.z:18789", 
      "token": "YOUR_SERVER_GATEWAY_TOKEN"
    }
  }
}
```

### 9.2 SSH 快捷连接配置
在 **本地 Mac** 编辑 ，简化服务器登录流程：
```text
Host nate-vps
    HostName 100.x.y.z     # 替换为服务器的 Tailscale IP
    User root             # 替换为您的服务器用户名
    IdentityFile ~/.ssh/id_rsa
```
*之后只需执行 `ssh nate-vps` 即可通过加密隧道安全登录。*

### 9.3 启动本地节点并挂载至云端
在 **本地 Mac** 终端运行，实现硬件能力与文件系统的远程共享：
```bash
openclaw run --node --node-name nate-mac --url ws://100.x.y.z:18789 --token <YOUR_SERVER_TOKEN>
```

### 9.4 架构收益
* **固定寻址**：无论服务器或本地 Mac 网络环境如何变换，Tailscale IP 保持永久不变。
*   **安全加固**：建议关闭云服务器防火墙的公网 18789 及 22 端口，仅保留内网/隧道访问。
